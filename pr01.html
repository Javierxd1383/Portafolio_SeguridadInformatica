1. INTRODUCCIÓN

Objetivo General:

Demostrar la capacidad de aplicar conocimientos teóricos de seguridad informática en un entorno práctico, mediante la realización de un walkthrough detallado y la explicación de las técnicas y metodologías utilizadas para vulnerar una máquina virtual.

Objetivos Específicos:

  • Configurar un entorno de virtualización aislado para realizar las pruebas de penetración de forma segura.
  • Utilizar herramientas de escaneo de puertos y servicios para identificar posibles vectores de ataque.
  • Enumerar los servicios y aplicaciones en ejecución para obtener información detallada sobre el sistema objetivo.
  • Investigar las vulnerabilidades conocidas asociadas con los servicios identificados.
  • Evaluar el riesgo y la viabilidad de explotar cada vulnerabilidad.
  • Seleccionar y adaptar los exploits adecuados para las vulnerabilidades encontradas.
  • Escalar privilegios para obtener acceso administrativo.
  • Registrar cada paso del proceso, incluyendo comandos utilizados, resultados obtenidos y capturas de pantalla.
  • Explicar las técnicas y metodologías utilizadas en cada etapa del ataque.
  • Grabar un video de la pantalla durante el proceso de ataque.
  • Explicar cada paso del proceso en detalle, utilizando un lenguaje claro y conciso.
  • Definir y explicar los términos técnicos utilizados durante el walkthrough.

3. DESARROLLO TÉCNICO

El proyecto busca, en primer lugar, preparar un entorno de pruebas seguro, seleccionando y configurando una máquina virtual vulnerable adecuada. A continuación, se realizará un reconocimiento exhaustivo del sistema objetivo, escaneando y enumerando los servicios en ejecución para identificar posibles vectores de ataque.

Se analizarán las vulnerabilidades presentes, investigando las conocidas y evaluando el riesgo de su explotación. Posteriormente, se procederá a explotar las vulnerabilidades identificadas para obtener acceso al sistema. Una vez obtenido el acceso, se realizarán actividades de post-explotación para demostrar el control del sistema, estableciendo persistencia y escalando privilegios.

Fases de Ejecución

1 RECONOCIMIENTO Y PREPARACIÓN DEL ENTORNO

Configuración de VirtualBox/VMware con la máquina SnakeOil (Digitalworld.local) en red NAT-only. Validación de conectividad y confirmación del objetivo.

VirtualBox / VMware nmap arp-scan netdiscover

2 ESCANEO DE PUERTOS Y ENUMERACIÓN DE SERVICIOS

Identificación de puertos abiertos, versiones de servicios y sistema operativo mediante escaneo activo.

nmap -sV -sC -O -p- --min-rate 5000 <IP_OBJETIVO>
nmap -sU --top-ports 100 <IP_OBJETIVO>
nmap whatweb nikto gobuster

3 ANÁLISIS DE VULNERABILIDADES

Investigación de CVEs asociados a los servicios identificados. Evaluación de vectores de acceso inicial.

searchsploit Metasploit CVEdetails NVD NIST

4 EXPLOTACIÓN

Selección y adaptación del exploit adecuado. Obtención de acceso inicial al sistema objetivo.

# Ejemplo de uso con Metasploit:
msfconsole
use exploit/[modulo_identificado]
set RHOSTS <IP_OBJETIVO>
run
Metasploit Burp Suite SQLmap Hydra

5 POST-EXPLOTACIÓN Y ESCALADA DE PRIVILEGIOS

Demostración del control del sistema, establecimiento de persistencia y escalada hasta tener acceso root/administrador.

# Verificar usuario actual y permisos
id && whoami
sudo -l
find / -perm -4000 -type f 2>/dev/null # SUID binaries
cat /etc/crontab # Cron jobs
LinPEAS GTFObins Meterpreter

Registro detallado de cada paso con capturas de pantalla, comandos y resultados. Grabación del video explicativo con narración de cada técnica utilizada.

5. CONCLUSIÓN

Este proyecto integra de manera práctica todos los conocimientos adquiridos a lo largo del curso de Seguridad Informática. La máquina virtual SnakeOil de VulnHub ofrece un entorno realista para aplicar la metodología completa de pentesting, desde el reconocimiento inicial hasta la escalada de privilegios.

El proceso demuestra que la seguridad de un sistema no depende únicamente de firewalls o antivirus, sino de una configuración correcta de los servicios, actualizaciones periódicas y el principio de mínimo privilegio. Cada vulnerabilidad encontrada representa una lección sobre cómo mejorar la postura de seguridad de sistemas reales.

6. REFLEXIÓN TÉCNICA DEL ESTUDIANTE

OPINIÓN PERSONAL:

Realizar este walkthrough me ha permitido entender que el pentesting es mucho más que ejecutar herramientas automáticas. Requiere pensamiento crítico y lateral: saber interpretar la información obtenida, encadenar vulnerabilidades aparentemente pequeñas para lograr un impacto mayor y documentar cada decisión de forma que pueda reproducirse y entenderse.

Lo que más me sorprendió fue la fase de escalada de privilegios. Los mecanismos como los SUID binaries o las cron jobs mal configuradas demuestran que un atacante con acceso limitado puede convertirse en superusuario si el sistema no está correctamente endurecido (hardening).

Vinculando con la práctica profesional: los administradores de sistemas que conozcan estas técnicas pueden proactivamente revisar sus propios sistemas usando herramientas como LinPEAS, Lynis o OpenVAS para identificar configuraciones peligrosas antes de que un atacante real las explote.
